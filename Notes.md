1. Basic SQL Commands
SELECT → Retrieve data from a table
INSERT → Add new records
UPDATE → Modify existing data
DELETE → Remove records
WHERE → Filter rows based on conditions
2. Data Types
Define the kind of data stored in columns
Examples:
INT → integers
VARCHAR → strings
DATE → date values
BOOLEAN → true/false
3. Constraints
Rules applied to columns
Types:
PRIMARY KEY → unique + not null
FOREIGN KEY → links tables
UNIQUE → no duplicates
NOT NULL → cannot be empty
CHECK → condition validation
4. Joins
Combine data from multiple tables
Types:
INNER JOIN → matching rows only
LEFT JOIN → all left + matched right
RIGHT JOIN → all right + matched left
FULL JOIN → all rows from both tables
5. Aggregate Functions
Perform calculations on data
Examples:
COUNT() → number of rows
SUM() → total
AVG() → average
MAX() / MIN() → highest/lowest
6. GROUP BY & HAVING
GROUP BY → group rows by column
HAVING → filter grouped data (after aggregation)
7. Subqueries
Query inside another query
Used for filtering, comparison, or data retrieval
8. Indexes
Improve query performance
Faster data retrieval but slower insert/update
9. Views
Virtual tables based on queries
Simplify complex queries
Do not store actual data
10. Stored Procedures
Predefined SQL code saved in database
Can be reused and executed when needed
11. Triggers
Automatically execute when events occur
Example: after insert, update, delete
12. Normalization
Organizing data to reduce redundancy
Forms:
1NF, 2NF, 3NF
13. Transactions
Group of SQL operations
Properties (ACID):
Atomicity
Consistency
Isolation
Durability
14. Keys
Identify records uniquely
Types:
Primary Key
Foreign Key
Candidate Key
Composite Key
